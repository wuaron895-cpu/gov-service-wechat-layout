# 排版方法论 SOP（风格复刻 5 步）

> 把任意政务号的排版风格，拆成可复用的模板 spec，沉淀为账号预设。
> 方法论源自「风格复刻工作流 SOP」，适配公众号排版场景。

## 5 步标准流程

### STEP 1 — 锁定参考源
- 挑 1–3 篇风格统一的参考推文（同账号、同类内容）
- 记录：账号、内容类型、最想复刻的结构/组件
- 抓取原始 HTML（公众号文章可通过 `curl mp.weixin.qq.com/s/<id>` 获取），用 grep 提取内联 `color:`/`background:` 量真实色值

### STEP 2 — 拆解维度
对每篇参考，按下表逐项拆解：

| 维度 | 字段 | 说明 |
|---|---|---|
| 头图 | header | 封面图 + 标题压字样式；文首是否引导关注 |
| 配色 | palette | 主色 / 强调色 / 背景色 / 文字色 HEX（量真实值，勿凭感觉） |
| 标题层级 | typeScale | 主标题 / 小节标题 / 正文 / 注释 字号行高 |
| 分隔 | divider | 模块之间用什么线 / 留白 / 序号块 |
| 信息卡 | callout | 温馨提示 / 误区框 / 案例框 / 举个例子 样式 |
| 文末引导 | footer | 咨询话术 / 二维码 / 署名 / 阅读原文 |
| 字体 | fonts | 中文无衬线为主，关键词加粗 |

### STEP 3 — 编码为 spec
把拆解写进本 skill 的 references：
- 配色 → `color-slot-model.md`（主题色插槽 + 候选库）
- 组件/字号 → `component-library.md`
- 骨架 → 对应内容类型模板 `assets/template-*.md.html`

### STEP 4 — 应用 + 校验
1. 用模板出新内容（替换 `{{…}}` 占位 + 套主题色）
2. 与参考推文并排 A/B：结构对不对、组件全不全、色准不准
3. 迭代 spec（通常 2–3 轮收敛）

### STEP 5 — 沉淀为预设
- 新内容类型 → 新增 `assets/template-*.md.html`
- 新主题色 → 加入 `color-slot-model.md` 候选库
- 从此一句话切类型/换色，跨文章复用 = 账号一致性

## 常见坑

- ❌ 混搭多风格 → ✅ 一次只复刻一种调性，收敛后再混
- ❌ 不落模板，每次重排 → ✅ 必须沉淀模板，否则白做
- ❌ 色值凭感觉 → ✅ grep 原始 HTML 量真实 HEX
- ❌ IP 口吻忽有忽无 → ✅ 全号统一一个拟人 IP

## 扩展到新账号

本 skill 模板已去标识化。给新账号套用时：
1. 跑 STEP 1–2 量该账号真实色值与组件
2. 把 `{{IP名}}/{{城市}}/{{热线}}/{{公众号名}}/{{机构全称}}` 替换为该账号值
3. 若该账号内容类型超出 6 类，按 STEP 5 新增模板
